中建三局资产管理系统

数据库结构说明书
技术栈： Vue3 + SpringBoot + MySQL + Redis
数据库版本： MySQL 8.0+
表前缀： zjsj_
文档版本： 1.0

一、数据库设计概述

本数据库设计服务于中建三局资产管理系统，围绕资产全生命周期管理构建核心数据模型，涵盖资产台账、租赁管理、周转物资、实时盘点等关键业务模块。

设计原则

  • 模块化设计：按业务功能划分表结构
  • 审计追踪：所有表均包含创建/更新信息
  • 空间数据处理：使用MySQL GIS扩展存储地理位置
  • RFID集成：专设物联网设备管理表
  • 命名规范：统一前缀 zjsj_

审计字段规范

所有表均包含以下审计字段，实现全系统操作追踪：

  • create_by - 创建者
  • update_by - 更新者
  • create_time - 创建时间
  • update_time - 更新时间

1. 资产主表 (zjsj_asset)

字段名 数据类型 约束 说明
id BIGINT PK, AUTO_INCREMENT 资产唯一ID
asset_no VARCHAR(50) NOT NULL, UNIQUE 固定资产编号(RFID/UUID)

update_time DATETIME DEFAULT NULL 更新时间

update_by VARCHAR(255) DEFAULT NULL 更新者

3. 无人机巡检表 (zjsj_drone_inspection)

update_by VARCHAR(255) DEFAULT NULL 更新者

四、数据库实现建议

1. GIS扩展支持

启用MySQL地理空间扩展：

INSTALL PLUGIN spatial SONAME 'ha_spatial.so';
CREATE SPATIAL INDEX idx_location ON zjsj_project(location);
CREATE SPATIAL INDEX idx_location ON zjsj_storage_area(location);

2. 审计字段触发器

创建通用更新触发器：

DELIMITER //
CREATE TRIGGER zjsj_update_timestamp
BEFORE UPDATE ON zjsj_asset
FOR EACH ROW
BEGIN
    SET NEW.update_time = NOW();
END//
DELIMITER ;

五、系统功能与表映射

核心功能数据支撑

系统功能 主要数据表 扩展支持
资产台账清单 zjsj_asset, zjsj_asset_category 资产状态统计、价值分析

审计追踪机制

所有核心业务操作均通过审计字段实现完整追踪：

  • 资产状态变更（入库/出库/维修）
  • 租赁订单审批流程
  • 巡检任务执行记录
  • 系统配置变更历史
